Programming Fluency & Language Confidence

Stop Googling "python for loop" ‚Äî build real fluency

The Real Problem

You can write scripts that work. But you feel like an imposter because:

  • You constantly look up basic syntax
  • You don't know if there's a "better" way to do something
  • Reading other people's code feels like reading a foreign language
  • You're not sure when to use a list vs. dict vs. set

This module fixes that. We'll build fluency ‚Äî the ability to express your ideas in code without friction.

Part 0: From Thinking to Code

Module 0 gave you a thinking framework. Now let's see how it translates to actual code. Here's a worked example showing the full process.

üìã Problem

Given a list of server logs, find the IP address that made the most requests.

logs = [
    "192.168.1.1 GET /api/users",
    "10.0.0.5 POST /api/login",
    "192.168.1.1 GET /api/products",
    "192.168.1.1 DELETE /api/users/5",
    "10.0.0.5 GET /api/users"
]

Step 1: The Five Questions

Inputs: A list of strings, each starting with an IP address.

Outputs: A single IP address (string) ‚Äî the most common one.

By hand: Go through each line, tally IP addresses, find the highest count. 192.168.1.1 appears 3 times, 10.0.0.5 appears 2 times. Answer: 192.168.1.1

Edge cases: Empty list? All different IPs (tie)? Malformed lines?

Simplest version: Assume valid data, ignore ties for now.

Step 2: What Shapes?

Breaking down what we did by hand:

  1. Transform ‚Äî Extract IP from each line
  2. Accumulate/Group ‚Äî Count occurrences of each IP
  3. Search ‚Äî Find the one with highest count

Step 3: Write It

# Python
def most_common_ip(logs):
    # Transform: extract IPs
    ips = [line.split()[0] for line in logs]

    # Accumulate: count each
    counts = {}
    for ip in ips:
        counts[ip] = counts.get(ip, 0) + 1

    # Search: find max
    max_ip = None
    max_count = 0
    for ip, count in counts.items():
        if count > max_count:
            max_count = count
            max_ip = ip

    return max_ip
// Go
func mostCommonIP(logs []string) string {
    // Accumulate: count each IP
    counts := make(map[string]int)
    for _, line := range logs {
        ip := strings.Fields(line)[0]
        counts[ip]++
    }

    // Search: find max
    var maxIP string
    maxCount := 0
    for ip, count := range counts {
        if count > maxCount {
            maxCount = count
            maxIP = ip
        }
    }
    return maxIP
}

Notice how the code structure directly mirrors the shapes we identified. No magic ‚Äî just translating the manual process into syntax. This is what fluency looks like: the gap between "I know what to do" and "I can write it" becomes small.

1.3 Control Flow: If, For, While

These let you make decisions and repeat actions.

If statements (making decisions)

# Python
if status_code == 200:
    print("Success")
elif status_code == 404:
    print("Not found")
else:
    print(f"Error: {status_code}")
// Go
if statusCode == 200 {
    fmt.Println("Success")
} else if statusCode == 404 {
    fmt.Println("Not found")
} else {
    fmt.Printf("Error: %d\n", statusCode)
}

For loops (repeating actions)

# Python - iterate over items
for server in servers:
    deploy(server)

# Python - iterate with index
for i, server in enumerate(servers):
    print(f"{i}: {server}")

# Python - iterate over dict
for key, value in config.items():
    print(f"{key} = {value}")
// Go - iterate over slice
for _, server := range servers {
    deploy(server)
}

// Go - iterate with index  
for i, server := range servers {
    fmt.Printf("%d: %s\n", i, server)
}

// Go - iterate over map
for key, value := range config {
    fmt.Printf("%s = %s\n", key, value)
}

üí° The _ (underscore)

In both Python and Go, _ means "I don't care about this value". Use it when you need the loop but don't need the index or value.

1.4 Functions: Reusable Code Blocks

Functions let you name a piece of logic and reuse it. Good functions do ONE thing.

# Python
def get_healthy_pods(namespace):
    """Return list of healthy pod names in a namespace."""
    pods = kubectl_get_pods(namespace)
    return [p.name for p in pods if p.status == "Running"]

# Usage
healthy = get_healthy_pods("production")
// Go
func getHealthyPods(namespace string) []string {
    pods := kubectlGetPods(namespace)
    var healthy []string
    for _, p := range pods {
        if p.Status == "Running" {
            healthy = append(healthy, p.Name)
        }
    }
    return healthy
}

// Usage
healthy := getHealthyPods("production")

When to make a function:

  • You're copying and pasting code
  • A block of code does one logical thing
  • You want to give a name to a complex operation
  • You want to test something in isolation

Part 2: Error Handling

Things break. Good code handles it gracefully.

Python: Try/Except

try:
    response = requests.get(url, timeout=5)
    data = response.json()
except requests.Timeout:
    print(f"Timeout connecting to {url}")
    data = None
except requests.RequestException as e:
    print(f"Request failed: {e}")
    data = None

Go: Explicit Error Returns

response, err := http.Get(url)
if err != nil {
    log.Printf("Request failed: %v", err)
    return nil, err
}
defer response.Body.Close()

data, err := io.ReadAll(response.Body)
if err != nil {
    log.Printf("Failed to read body: %v", err)
    return nil, err
}

üîß Python vs Go Philosophy

Python: "Ask for forgiveness, not permission" ‚Äî try it and catch the error.
Go: "Errors are values" ‚Äî check every operation explicitly. This is why Go code has if err != nil everywhere.

Part 3: Reading Other People's Code

This is a skill. Here's how to build it:

Strategy 1: Start from main()

Find the entry point. In Python, look for if __name__ == "__main__":. In Go, find func main(). Start there and follow the calls.

Strategy 2: Ignore details first

Don't try to understand every line. First pass: What are the major functions? What do they seem to do based on their names?

Strategy 3: Find the data structures

What types/classes/structs are defined? These tell you what the code is modeling.

Strategy 4: Run it with print statements

Add prints/logs to understand the flow. There's no shame in this ‚Äî it's debugging.

‚ö†Ô∏è Common Patterns You'll See

  • *args, **kwargs (Python) ‚Äî "accept any arguments"
  • lambda x: x.name (Python) ‚Äî inline mini-function
  • [x for x in items if x.valid] ‚Äî list comprehension
  • interface{} or any (Go) ‚Äî "any type"
  • defer (Go) ‚Äî "run this when the function exits"

Part 4: Common Idioms

These patterns appear everywhere. Learn them once, recognize them forever.

Python Idioms

# List comprehension (filtering + transforming)
names = [pod.name for pod in pods if pod.status == "Running"]

# Dict comprehension
env_map = {k: v for k, v in os.environ.items() if k.startswith("APP_")}

# Get with default (never KeyError)
timeout = config.get("timeout", 30)

# Context manager (auto-cleanup)
with open("config.yaml") as f:
    config = yaml.safe_load(f)

# Join strings
hosts = ", ".join(server_list)  # "web1, web2, web3"

# F-strings (formatting)
message = f"Deployed {app} to {env} in {duration:.2f}s"

Go Idioms

// Short variable declaration
name := "nginx"  // instead of: var name string = "nginx"

// Check and assign in one line
if err := doThing(); err != nil {
    return err
}

// Defer for cleanup
f, _ := os.Open("config.yaml")
defer f.Close()

// Type assertion
if s, ok := value.(string); ok {
    fmt.Println(s)
}

// Zero values are useful
var count int    // automatically 0
var name string  // automatically ""
var ready bool   // automatically false

Exercises

Exercise 1.1: Type Recognition

For each piece of data below, decide: List, Dict, or Set?

  1. Kubernetes pod labels
  2. Unique error codes seen in the last hour
  3. Ordered list of deployment stages
  4. Mapping of service names to their ports
  5. All namespaces in a cluster
üí° Hint

Ask yourself: Do I need key-value pairs? Do I need ordering? Do I need uniqueness? Do I need fast lookup?

‚úÖ Solution
  1. Dict ‚Äî Labels are key-value pairs: {"app": "nginx", "env": "prod"}
  2. Set ‚Äî We want unique codes, don't care about order: {500, 502, 503}
  3. List ‚Äî Order matters: ["build", "test", "deploy", "verify"]
  4. Dict ‚Äî Mapping name to port: {"web": 80, "api": 8080}
  5. Set or List ‚Äî Set if you need fast "is this namespace present?" checks; List if you need to iterate in a specific order

Exercise 1.2: Translate Between Languages

Take this Python snippet and rewrite it in Go (or vice versa):

servers = ["web1", "web2", "web3"]
healthy = []
for server in servers:
    if check_health(server):
        healthy.append(server)
print(f"Healthy: {len(healthy)}/{len(servers)}")
üí° Hint

In Go: slices use append(), formatting uses fmt.Printf(), and you'll need to handle the error from check_health.

‚úÖ Solution (Go) 
servers := []string{"web1", "web2", "web3"}
var healthy []string
for _, server := range servers {
    if checkHealth(server) {
        healthy = append(healthy, server)
    }
}
fmt.Printf("Healthy: %d/%d\n", len(healthy), len(servers))

Key differences: explicit types, range keyword, append returns new slice.
